# Voice & Taste

*The most useful file in this folder for a collaborator. Read it before you generate anything that should sound like Nihar or sit inside his work.*

---

## The voice

Three words govern all copy: **precise, calm, human.**

- **Precise** — concrete over abstract. Names the actual intervention, not the grand mission. "I've found joy in breaking and making processes within live systems" — specific verbs, real stakes.
- **Calm** — no hype, no exclamation energy, no buzzwords. Long, settled sentences. Confidence shown by restraint, not volume.
- **Human** — warm, occasionally wry, occasionally vulnerable. *"Advanced Designer, Basic Conversationalist."* *"I live well to be able to design well."* It admits feeling without getting sentimental.

### Tells and tics (so generated copy rings true)
- Lowercase-y playfulness lands sparingly — e.g. "make them flowww" — a single deliberate stretch, never a habit.
- Em-dashes and short fragments for rhythm. "Small interventions. In work. In daily life."
- First person, direct address. He talks *to* the reader, invites them in: "Welcome to my collection of design interventions aka my portfolio."
- Wry self-awareness over self-promotion. Humor is dry and structural, never jokey.
- Metaphors come from physical craft and systems — paper, surfaces, rhythm, flow, mechanics — not from tech-marketing.

### What to avoid (anti-voice)
- Hype words: "revolutionary," "seamless," "cutting-edge," "passionate," "delight" as a noun.
- Abstraction-for-its-own-sake. If a line could be on any designer's site, it's wrong.
- Over-cleaning. Nihar's hand-authored quirks (odd spacing, a stretched word, an off-true rotation) are *intentional* — don't normalize them into generic polish.
- Recruiter-speak and bullet-list resume tone in prose contexts.

## The design philosophy

These are the principles the work is built on. They're as much about *how Nihar thinks* as about the site.

1. **Design is organized attention.** Every layout decision is a decision about where the eye goes and in what order. A page is authored, like an essay.

2. **Reading-first, not scroll-first.** The work draws on editorial practice — print case studies, architectural monographs, museum catalogs — far more than on SaaS portfolio templates. It wants to be read.

3. **The flatlay grammar.** Things are *placed*, not scattered. Slight rotation, soft shadow, breathing room. Intentional arrangement is the aesthetic.

4. **Docked, tucked, or suspended — never just nearby.** Relationships between elements are explicit and physical. Notes dock to the evidence they annotate. Cards stack. Reveals stay latent, then release.

5. **Proof artifacts stay proof artifacts.** Sketches, boards, mockups must keep reading as real evidence — never decorative placeholder.

6. **Controls must not lie.** If something looks interactive, it works — or it stops implying interaction.

7. **Paper-physical motion.** Everything glides, settles, lands. No bounce, no overshoot, no snapping. Calm by default.

8. **Small interventions in live systems.** The unifying worldview across graphic, product, and infrastructure work: improve the running system with deliberate, modest, well-placed changes.

## Materials & references (Nihar's taste, sourced)

The portfolio openly cites what shaped it — useful for understanding his references:

- **Typography:** Fraunces (display serif, the editorial voice), Google Sans / Google Sans Flex (body & UI — warm humanist, "authored, not defaulted"), Material Symbols Rounded (icons), a system monospace for timeline year labels.
- **Color:** OKLCH for perceptually-even project palettes; Munsell (via munsell.js) for the landing spectrum grid. Color science done offline — only final values ship.
- **Shadow & elevation:** semantic four-tier model after Atlassian (`flat → resting → raised → overlay`); shadow *restraint* after Radix — quiet shadows on warm paper, not loud ones on white.
- **Motion:** one easing curve as a philosophy, not a library. Framer Motion where physics help; the native Web Animations API for a page-transition ghost-clone; CSS for the rest.
- **Texture & shaders:** a slowed, softened GLSL port of localthunk's *Balatro* title background sits under the `/rr` Cards section; hand-milled paper grain via SVG turbulence.
- **Editorial lineage:** long-form print design more than any web template.
- **Collaborator:** Akshar Dave developed the original `/biconomy` page — the evidence-and-notes language and the sheet-stack model trace back to that work. Nihar names it as "a real debt, not a footnote."
- **Tools, by job:** ChatGPT for organizing thoughts and early drafts ("the part of the work that has no file yet"); Claude Code for the actual building; Figma for layout and palettes; v0 for prototyping the `/rr` game.

The distinction Nihar draws — *thinking tools vs. building tools* — is itself characteristic. He's deliberate about which instrument does which job.

## How to collaborate with him well

- He's a **designer, not a CTO**. Treat requests as design conversations. "Could we do X?" usually means "*is* X the right call?" — so evaluate and recommend, don't just execute.
- **Diagnose before fixing.** Find the root cause before proposing a change.
- **Recommend, don't hedge.** A clear point of view is partnership; a menu of options is not.
- **Default to the smallest correct change.** The portfolio is in a finishing stage — polish, not reinvention. Bring the same instinct to ideas: sharpen what exists before proposing net-new.
- **Surface adjacent ideas** worth considering — and say which to skip.
