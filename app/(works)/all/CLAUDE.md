## Don't-touch digest — bench essay (Work Essay)

- The ticket foots the invitation card and is SCROLL-coupled via `useBenchDock` (mirrors the nav cluster's `useDockedMarker`): native-scroll + ONE down-only assist. It returns **ONE state, `engaged`**, applied as `.is-pinned` (position) AND `.is-condensed` (visual condense) TOGETHER as a single unit — pin + condense fire at the same instant. `engaged` flips by READING the dock line only (`slot.top <= DOCK_TOP+1`, `DOCK_TOP=16` → `position:fixed; top:16`); native scroll stays in charge, never hijacked to flip it. NO scale (read as an odd enlarge). **The two states are intentionally merged — do NOT re-split into `pinned`/`condensed`.** SUPERSEDED + removed: the old "two-anchor midpoint commit" (Anchor 0/1, `SEAT` cushion, midpoint `scrollGlide`) fired programmatic scroll mid-gesture and fought native momentum; don't reintroduce it or the SEAT. See ANOMALIES → "Scroll-dock + shell contract".
- **The ONE assist: down only, finishes a committed descent.** On scroll-idle (`IDLE_MS`=150ms after the last scroll) the hook glides to `workY()` (= the `.bench-work` panel top — card fully cleared; NO SEAT, NO midpoint) ONLY IF: no glide active, `engaged` already true (ticket has docked), last scroll direction was DOWN, reduced-motion off, and not already at target. It never fires from the un-docked top (a peek is left to native scroll) and NEVER fires upward. Any in-flight glide ABORTS the instant the reader scrolls against it (`glideRef` {target, cancel}; a contrary scroll delta cancels it) — so it never fights. `glideTo`: down=`--dur-glide`, reverse/✕=`--dur-settle`, reduced-motion=instant `scrollTo`.
- `useBenchDock` pins the slot's `min-height` (rest footprint) so the card doesn't collapse / the work doesn't jump when the ticket pins out, and runs the clearPane effect. The rest→fixed flip is scroll-coincident (seamless); the navbar is centred via `left:50%` + a constant `transform:translateX(-50%)` (width-independent, so the shrink stays centred with no horizontal jump).
- **The whole broadside scales by `--card-scale` (0.75 on `.bench-stage`)** — a real LAYOUT scale (`calc(token * var(--card-scale))` everywhere), NOT `transform:scale` (which would make the card the containing block for the fixed docked ticket). Tune the knob, then adjust from there.
- **Condense — definite width that interpolates, flex centred, in-flow close.** Condense animates on **`--dur-slide`/`--ease-snap`** (was `--dur-settle`). REST width is DEFINITE `calc(460px * var(--card-scale))` + `max-width:100%` (NOT `width:100%`) — **this SUPERSEDES the old "max-width:460 is load-bearing / dock explodes the ticket" item; a definite px width can't explode, and px→px means width + height collapse in lockstep**. Condensed width is `min(290px, calc(460px * var(--card-scale)))` — a definite ~290 (the `min()` is an inversion guard so the condense can't GROW at small scale). 290 is FIXED, NOT scaled, because the tab sub-labels are a fixed 11px that don't scale — a scaled box clipped them; at 290 the content fits with comfortable margins (edges > inter-item gaps, by intent). **This SUPERSEDES the old even-gap/slack→0 construction — don't re-tune to even gaps.** Detail smoothing: bead FADES (opacity, not display:none); `.bench-perf__line` transitions `width` (4→1) with `height`; `.bench-perf` width is definite. Rest tabs are grid `1fr auto 1fr` (centres the diamond); CONDENSED tabs switch to centred flex — the grid→flex swap is a known one-frame rearrange; a JS FLIP to smooth it was tried + REVERTED (CSS can't without regressing the diamond or the tight fit), so it stays. Close is IN-FLOW (`position:static`) when condensed, absolute+`opacity:0` at rest. Reduced motion zeroes the condense transitions. Don't revert tabs to grid, don't make either width fit-content/%, don't pull the close out of flow when condensed. See ANOMALIES → "Condense".
- The condense only animates on REAL wheel/trackpad scroll; an instant programmatic `scrollTo` jump bypasses the CSS transition (mid-flight width measurements after a jump are unreliable — settled values are correct).
- **Press states are scoped to `.is-condensed` only** — `.bench-tab:active` (`scale(0.95)` + ink/underline deepen) and `.bench-ticket__close:active` (`scale(0.88)` + colour deepen), micro tier, no bounce/pill. Don't extend to the rest (invitation) state.
- **The docked ticket is `position:fixed`, so the containing-block guards are load-bearing:** `.transition-pane:has(.bench-workbench){will-change:auto}` in bench.css (the shared pane's `will-change:transform` would otherwise pin the navbar to the pane), AND the slide-in entrance uses `animation-fill-mode: backwards` (NOT `both`) so `.bench-card` retains no transform. Either retained transform on a ticket ancestor re-breaks the fixed navbar.
- Entry paths: raw scroll docks the ticket natively, then the idle-settle finishes the descent (down only — see the assist item above); a tab click (`openTab`) `glideTo(workY())` into the work only when still resting at the invitation (`!engaged`), else just swaps content; ✕ (`close`) is `glideTo(0, '--dur-settle')` back to the card. Reduced motion swaps the glide for an instant `window.scrollTo` (bench.css zeroes the condense transitions to match). Switching tabs while engaged keeps scroll position.
- Deep-link entry (`/cases`, `/showcase`, `/all?showcase`, `/all?cases`, case-study EXIT) reads the bare query flags (`'showcase' in sp` / `'cases' in sp`) SERVER-side in `page.tsx` (the rewrites deliver them as the destination query; client `useSearchParams` never sees them; showcase wins if both present) and selects the active tab. It does NOT auto-scroll into the work (rests at the card) — auto-scroll-into-content is a deferred follow-up.
- TransitionSlot's exit-dim selector includes `.bench-workbench > *` — keep the wrapper class or the ghost won't dim on exit.
- Longform tab hosts `SelectedContent` in `.bench-cases` (sized relative wrapper) because the timeline mat is `position:absolute right:0 width:688`; its archive-open growth (729→1201) is mirrored on `.bench-cases:has(.selected-mat--archive-open)`.
- Showcase row-spans are state-driven (`--sc-rowspan` from React state in `Showcase.tsx`, not imperative setProperty — re-renders would wipe an imperative value); the rAF cleanup must null `rafRef` (a stale id makes the next measure bail forever under Strict Mode). Fallback span is 18 (≈756px), not 30. Measurement uses `offsetHeight` (transform-immune), NOT `getBoundingClientRect().height`, so the filter recede's slot scale never re-packs the grid.
- **Visual-tab category filter (FilterStrip): All · Interface · Brand.** A single-select `radiogroup` (NOT tabs/tablist — it filters one grid in place; a tablist would nest a 2nd competing tablist under the ticket's real tabs). Recede/emphasis is PURE CSS (no JS choreography): React flips `data-matched` (per `.sc-slot`) + `.is-filtering` (on `.sc-grid`); the SCALE lives on `.sc-slot` (not `.sc-piece`, so it never fights the tile rotate/hover). Matched→`scale(--sc-matched-scale)`; receded→`scale(--sc-recede-scale)` + `--backseat-dim` monochrome. ONLY the step-back uses `--sc-ease-recede` (a route-local SANCTIONED overshoot — the one no-bounce deviation here). Receded tiles stay CLICKABLE (a click = WAAPI "no" shake, doesn't open) and force-PAUSE media; the open tile auto-closes if filtered out. Filter state lives in `WorkPanel` (survives tab switches); categories = `PIECE_CATEGORY` in `data.ts`. Don't convert to tabs. See ANOMALIES → "Filter strip".
- Showcase tile caption (`.sc-cap`) shows the dot + `piece.type` + `piece.year` ONLY — no project name (it lives in the SpecNote). Reading order is by `num` (1→10); the RR card (cardstack) ↔ Aleyr (furrmark) `num`s are swapped (cardstack leads, furrmark in row 3). Menu-open recede dims the other tiles to MONOCHROME via the shared `--backseat-dim` filter (`.sc-grid.is-dimming`), not an opacity/saturate fade. SpecNote resting rotation is now always 0° (`--sc-note-rotate` left unset; the per-mount random ±2° roll was removed). See ANOMALIES → "Caption content" / "Bento reading order" / "Click + focus interaction" / "Spec note".

## Don't-touch digest — timeline + archive (the Longform tab content)

- Hover dim/highlight uses `filter: opacity()`, never the `opacity` property — Framer Motion writes inline opacity after entrance animations and silently overrides CSS opacity rules.
- The card-hover cascade is gated on `data-armed="true"`, set by ProjectCard.tsx only after the first real cursor mousemove; all five `:has()` cascade selectors plus the mobile undo rule are a unit — never drop the gate from any one of them.
- `data-armed` must live on the inner `<Link>`, not the outer motion.div — the `:has()` selectors target that element; timer-based and outer-element gates were tried and failed.
- All bars grow via `scaleY` with `transform-origin: top center` on the base bar class — do not override per-bar.
- Year labels use `transform: translate(-100%, -50%)`, so their CSS `top` is the visual center — every position calculation must account for this.
- The `D` delay map (Timeline.tsx) is absolute values, not relative offsets — changing any timing requires recalculating every delay after it. The three case studies have a SEPARATE `CHILD_D` expand stagger (relative to the dropdown opening).
- **Slangbusters is a second tall parent (mint) below Biconomy.** The inline "Slangbusters case studies (3)" dropdown (a text button on the mint spine, not a NavMarker) toggles `expanded`, which grows the mint bar (224→504) + the mat and shifts the lower timeline (mint card, nameplates) DOWN 280px — those animate `top`/`height` on `--dur-settle`, together with the mat `min-height`.
- The "mat grown" modifier is still named `.selected-mat--archive-open` (historical name kept). Its heights — **958 collapsed → 1238 open** — are mirrored on `.bench-cases:has(.selected-mat--archive-open)` in **bench.css**. Change BOTH together or the Cases tab clips.
- The three nested children (`.sb-case` — Aleyr/Ecochain/Codezeros) mount only when expanded (Framer `AnimatePresence`), at a 96px vertical pitch. The `CHILDREN` array in Timeline.tsx is the year source of truth — the hand-placed year labels must agree with it.
- Entrance motion is always top-to-bottom (`y: -8 → 0`); never a positive initial y. Dots pop (SPRING_POP, staggered), never fade. Year labels stay in `--font-mono`. The 12px dot-cluster grid + 4px label clearances are authored constants.
- Icon arrow hover animations are CSS-only translate on the inner SVG group (`app/components/icons/`) — no Framer Motion.
- The "opens in new tab" hint pill (mint card + `.sb-case`) keeps a neutral grey shell with only the text color themed per project — don't theme the shell.
- **Ecochain's hover bar fill (`--ecochain-240`) is a saturated green, NOT the old off-white** — the off-white read as invisible on the mat; the bar must stay legible.
- The Now dot's living pulse is a separate `.selected-tl__pulse` sibling ring + `now-pulse` keyframe — NOT a pseudo, so it never collides with the dot's morning `clip-path` / evening `::before` crescent. Honors reduced-motion (own guard, since the route's old `.selected-workbench` reduced-motion block is stale post-rename).
- **Mobile renders a separate composition** (`MobileCases` — cards-first, with the case studies in `CasesSheet`, a bottom sheet) behind a `matchMedia(MOBILE_BP)` gate in SelectedContent — NOT a CSS reflow of the desktop rail. The two never coexist in the DOM (no duplicated content). `MOBILE_BP` is the single source in `Showcase/responsive.ts`.
- `CasesSheet` is rendered INLINE (not portaled to `<body>`) so the route-scoped study tokens (`--aleyr/ecochain/codezeros-*`, defined on `.workbench:has(.bench-workbench)`) cascade in. `position: fixed` still anchors to the viewport — no transformed ancestor in the cases tree (the same guarantee the docked ticket relies on).
- selected.css's `@media` tablet block was removed (stale): on bench, `.bench-cases` centres the 688 mat at tablet, so the desktop layout is already correct down to ~704px. The old block repositioned a `.selected-layout` that no longer exists and squished the mat.
