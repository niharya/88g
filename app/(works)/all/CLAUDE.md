## Don't-touch digest — bench essay (Work Essay)

- The ticket foots the invitation card and is SCROLL-coupled via `useBenchDock` (mirrors the nav cluster's `useDockedMarker`): native-scroll + ONE down-only assist. It returns **ONE state, `engaged`**, applied as `.is-pinned` (position) AND `.is-condensed` (visual condense) TOGETHER as a single unit — pin + condense fire at the same instant. `engaged` flips by READING the dock line only (`slot.top <= DOCK_TOP+1`, `DOCK_TOP=16` → `position:fixed; top:16`); native scroll stays in charge, never hijacked to flip it. NO scale (read as an odd enlarge). **The two states are intentionally merged — do NOT re-split into `pinned`/`condensed`.** SUPERSEDED + removed: the old "two-anchor midpoint commit" (Anchor 0/1, `SEAT` cushion, midpoint `scrollGlide`) fired programmatic scroll mid-gesture and fought native momentum; don't reintroduce it or the SEAT. See ANOMALIES → "Scroll-dock + shell contract".
- **The ONE assist: down only, finishes a committed descent.** On scroll-idle (`IDLE_MS`=150ms after the last scroll) the hook glides to `workY()` (= the `.bench-work` panel top — card fully cleared; NO SEAT, NO midpoint) ONLY IF: no glide active, `engaged` already true (ticket has docked), last scroll direction was DOWN, reduced-motion off, and not already at target. It never fires from the un-docked top (a peek is left to native scroll) and NEVER fires upward. Any in-flight glide ABORTS the instant the reader scrolls against it (`glideRef` {target, cancel}; a contrary scroll delta cancels it) — so it never fights. `glideTo`: down=`--dur-glide`, reverse/✕=`--dur-settle`, reduced-motion=instant `scrollTo`.
- `useBenchDock` pins the slot's `min-height` (rest footprint) so the card doesn't collapse / the work doesn't jump when the ticket pins out, and runs the clearPane effect. The rest→fixed flip is scroll-coincident (seamless); the navbar is centred via `left:50%` + a constant `transform:translateX(-50%)` (width-independent, so the shrink stays centred with no horizontal jump).
- **The whole broadside scales via container-query units (`--bu`) — PURE CSS, no JS** (the old `--card-scale` JS resize handler is removed). The card is a locked 3:4 page sized off viewport height — sibling of the landing's framed sheet. Two halves: (1) **width** = `--bench-card-w` on `.bench-stage` = `min(clamp(508px, (100svh−104)/1.3333, 615px), 100vw − 2*var(--sheet-gutter))`, `max-width`'d onto the stage; `.bench-card { width:100%; aspect-ratio:3/4 }` gives the height. (2) **content** = `.bench-stage` is the query container (`container-type: inline-size`) with `--bu: calc(100cqi / 820)` ("one baseline pixel"); every card+ticket size is `calc(N * var(--bu))`. **The container is the STAGE, not `.bench-card`** — the stage has no border/padding so cqi resolves against the clean card width; on the card it's circular (cqi → content box minus the card's own `--bu` padding → resolves too small). `--bu` = 0.75px when the card is 615 wide → reproduces the old render exactly; floor 508 then scroll. **The cap is RELATIVE: `min(80vw, 820px)`** (not a fixed 615) — on portrait/tablet screens (3:4, like iPads) the card nests concentrically leaving a consistent ~10% side margin at every size (a fixed 615 froze it small on bigger iPads: Pro 12.9 fell to 60% of width). 80vw keeps the ~80% nesting; the `820px` ceiling = the scale-1 baseline (`--bu` resolves to 1px → the card renders at its true authored size). Self-targets to portrait — in landscape/desktop the height term is always < 80vw so the proportional cap never binds; desktop is unchanged until a window taller than ~1197px svh (then the card reaches full scale-1 size). NOT `transform:scale` (would trap the fixed ticket); `container-type: inline-size` does NOT trap it (verified — pins to viewport, `--bu` resolves to the stage width even on the fixed ticket). `.bench-card__main` is `flex:1; justify-content:center` (ticket slot's pinned min-height keeps docking from recentring). Tune via the `--bench-card-w` clamp (floor/cap/`CHROME 104`) on `.bench-stage`. See ANOMALIES → "Viewport-driven 3:4 card" + "`--bu` container-query spine".
- **Condense — definite width that interpolates, flex centred, in-flow close.** Condense animates on **`--dur-slide`/`--ease-snap`** (was `--dur-settle`). REST width is DEFINITE `calc(460 * var(--bu))` + `max-width:100%` (NOT `width:100%`) — **this SUPERSEDES the old "max-width:460 is load-bearing / dock explodes the ticket" item; a definite px width can't explode, and px→px means width + height collapse in lockstep**. Condensed width is `min(236px, calc(460 * var(--bu)))` — a definite ~236 (the `min()` is an inversion guard so the condense can't GROW at small scale). 236 is FIXED, NOT scaled, because the tab sub-labels are a fixed 10px that don't scale — a scaled box clipped them; at 236 the whole row (tabs · divider · in-flow ✕) fills the box with tight, balanced ~23px margins on even inter-item gaps (`20 * var(--bu)`). **Retuned down from 290, which read too loose / left-heavy with a floating ✕ — so the even-gap tight row is now the target (this reverses the old "comfortable wide margins, don't re-tune to even gaps" note).** Detail smoothing: bead FADES (opacity, not display:none); `.bench-perf__line` transitions `width` (4→1) with `height`; `.bench-perf` width is definite. Rest tabs are grid `1fr auto 1fr` (centres the diamond); CONDENSED tabs switch to centred flex — the grid→flex swap is a known one-frame rearrange; a JS FLIP to smooth it was tried + REVERTED (CSS can't without regressing the diamond or the tight fit), so it stays. Close is IN-FLOW (`position:static`) when condensed, absolute+`opacity:0` at rest. Reduced motion zeroes the condense transitions. Don't revert tabs to grid, don't make either width fit-content/%, don't pull the close out of flow when condensed. See ANOMALIES → "Condense".
- The condense only animates on REAL wheel/trackpad scroll; an instant programmatic `scrollTo` jump bypasses the CSS transition (mid-flight width measurements after a jump are unreliable — settled values are correct).
- **Press states are scoped to `.is-condensed` only** — `.bench-tab:active` (`scale(0.95)` + ink/underline deepen) and `.bench-ticket__close:active` (`scale(0.88)` + colour deepen), micro tier, no bounce/pill. Don't extend to the rest (invitation) state.
- **The docked ticket is `position:fixed`, so the containing-block guards are load-bearing:** `.transition-pane:has(.bench-workbench){will-change:auto}` in bench.css (the shared pane's `will-change:transform` would otherwise pin the navbar to the pane), AND the slide-in entrance uses `animation-fill-mode: backwards` (NOT `both`) so `.bench-card` retains no transform. Either retained transform on a ticket ancestor re-breaks the fixed navbar.
- Entry paths: raw scroll docks the ticket natively, then the idle-settle finishes the descent (down only — see the assist item above); a tab click (`openTab`) `glideTo(workY())` into the work only when still resting at the invitation (`!engaged`), else just swaps content; ✕ (`close`) is `glideTo(0, '--dur-settle')` back to the card. Reduced motion swaps the glide for an instant `window.scrollTo` (bench.css zeroes the condense transitions to match). Switching tabs while engaged keeps scroll position.
- **The Visual↔Longform tab swap animates** (`WorkPanel.tsx`): `AnimatePresence mode="wait"` keyed on `active`, riding the SHARED tab-switch idiom from `app/lib/motion.ts` (`TAB_BODY_VARIANTS` + `TAB_BODY_TRANSITION`) — the same tokens `/rr` + `/biconomy` use, NOT a new tier. `initial={false}` skips the first-mount wipe so a deep-link/reload lands at rest. The body's `scale:0.985→1` is SAFE because it only paints during the switch + settles to `transform:none` (never a lasting containing block) AND `WorkPanel` is a SIBLING of the docked ticket (not an ancestor — can't trap its `position:fixed`) AND the showcase grid measures via `offsetHeight` (transform-immune — the enter-scale can't re-pack it). Don't move the scale onto a ticket ancestor; don't switch the row-span measure to `getBoundingClientRect().height`. See ANOMALIES → "Tab swap animates".

## Don't-touch digest — timeline + archive (the Longform tab content)

- **The timeline is a FLOW layout, not absolute coordinates.** `.selected-tl` is a flex column of blocks: a cap (`.selected-tl__cap`), dot clusters, two project GROUPS, and the nameplates. Heights are content-driven; **do NOT reintroduce absolute `top` coordinates for the body**.
  - **Biconomy group = a 2-ROW × 2-COL grid** (`.selected-tl__group--blue`). Bars, years, and cards are DIRECT grid items (no rail/cards wrapper cells). Cards in col 2 (RR row 1, Biconomy row 2). Bars in col 1 are grid items `justify-self:end; align-self:stretch` so each is EXACTLY as tall as its card row(s): yellow `grid-row:1` (spans the Rug Rumble card, edges touching), blue `grid-row:1/3` (spans both). Years are col-1 grid items aligned to their card's TOP (2025, `align-self:start`) or ROLE ROW (Q4•25 + 23, `align-self:end` + a `margin-bottom` lift onto the footer line).
  - **Slangbusters group** (`.selected-tl__group--mint`) is a flex column: the dropdown HEADER on top, then `.selected-tl__mint-spinebox` (a relative flex column of rail+card ROWS — parent + children). The mint spine is one absolute bar over the SPINEBOX (not the header), so it starts at the Slangbusters CARD top and grows to span the children when they mount. Mint-rail years are absolute within their row's rail.
- **The mat is a framed sheet — sibling of the landing sheet + invitation card.** `.selected-mat` carries the two-layer lift shadow (green-tinted `rgba(10,48,24,…)`) over its thin `mint-100` frame (`calc(1 * var(--bu))` — reduced from 2·--bu to match live); the framed-sheet read comes from the LIFT, not a heavy rule (deep `mint-720` + a ~5px inset band were both tried and rejected).
