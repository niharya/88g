## Startooth engine vocabulary

**Voids** — `star` (8-sided) and `diamond` (4-sided, below the star). `isVoid(u)` checks both.
**Keys** — `shortkey` (upper, between two stars; FACES_SHORTKEY + TOP_SHORTKEY) and `tallkey` (lower, between star and diamond; FACES_TALLKEY + TOP_TALLKEY). `isKey(u)` checks both. Key parts: **faces** (left+right panels) and **top** (protruding cap, distinctly coloured).
**Static data** — all geometry, palette, and timing live in `startooth-constants.ts`. `StartoothField` holds them as `private readonly` aliases — method bodies are unchanged. Editing a constant: change it in `startooth-constants.ts` only.

## Don't-touch digest

- Two motion idioms must never mix: Group A (about-short only) hero-tucks via top/scale transitions; Group B (about-long, spectrum, about-practice, contact) settles from above via opacity + translateY — about-practice is Group B not an extension of about-long even though the copy continues, and about-long is Group B (migrated from A in the practice-timeline redesign — too tall to tuck), with `.about-card--long { animation: none }` opting out of the `hero-glide-up` mount keyframe.
- `--stack-stagger-start` is tuned against Group A's tuck-out duration; lowering it makes the two groups visually compete.
- Group B cascade delays are sequential (spectrum +0s, about-practice +0.02s, contact +0.04s, about-long +0.06s) — about-long's delay was appended at +0.06s even though it's the first Group B slot positionally; any new card continues the cascade, never restarts at 0.
- Every Group B card in `.landing--default` must hold all four properties: top at its final expanded position, opacity 0, pointer-events none, and the `translateY(calc(-1 * var(--stack-settle)))` offset — dropping any one breaks the settle-from-above physics.
- Never re-anchor spectrum or contact's collapsed state to `--hero-top` or reintroduce `scale()` on them — that recreates the ghost-cards-descending bug.
- Group B cards rest at 0deg; reintroducing random rotation requires BOTH restoring `@property <angle>` typing AND batching the reroll with `setExpanded` in the same React commit.
- Spectrum hover/press (it's interactive — cycles the palette): hover (desktop-gated `@media (hover:hover) and min-width:768 and min-height:501`) lifts `.spectrum__frame` `translate: 0 -2px` + `--shadow-raised` on `--dur-slide`/`--ease-paper`; press `.spectrum--pressed .spectrum__frame { scale: 0.99 }` via `specPressed` state on `--dur-instant`/`--ease-snap`. Frame carries a transparent base shadow so the hover shadow can fade in (box-shadow can't interpolate from `none`). Lift/press use standalone `translate:`/`scale:` NOT `transform` so they compose with the click reroll's `transform: rotate()` (rr StoryCard idiom) — never fold them into `transform`. `.spectrum:hover` also strengthens the gradient bar `.spectrum__gradient` (opacity 0.1→0.32, top-stop `--orange-800` mix ~5%→~26%) on `--dur-slide`/`--ease-paper` — keep the `transition` on the base `.spectrum__gradient` rule or it snaps.
- Spectrum's artistic tilt is scroll-driven, not static: `--spec-scroll-tilt` on `.spectrum` (`rotate()`, no transition) eases 0°→−1° over the first 120px of scroll via a rAF window-scroll `useEffect` in page.tsx, frozen at 0° under reduced motion; the old static `rotate(-1deg)` was removed from all four `.landing__section--spectrum` rules. Framer Motion (`useScroll`/`useSpring`) was tried here and its motion values were inert — don't modernize back to it. (Click reroll `specRotation` rides `.spectrum__frame` and composes on top.)
- On expand, each Group B card's top, opacity, and transform transition together with the same duration and delay on desktop — never stagger the three within one card. (Documented mobile deviation: spectrum's transform rides `--dur-settle` +0.12s — shipped and authored; don't "fix" it or copy it to other cards.)
- The form-open height bump (`:has(.contact-card--form-open)`, +600px) must be remeasured at all breakpoints whenever `--contact-top`, the contact card's collapsed height, or the form's reveal target changes.
- DESKTOP cascade is SELF-ADJUSTING: only `--hero-top` + `--hero-h` are free measurements; `--projects-top`/`--long-top`/`--spectrum-top`/`--practice-top`/`--contact-top`/`--expanded-h` are a `calc()` chain off the settled hero bottom (`--hero-top + --hero-h`), each offset = the block above's height + gap. Changing the hero height re-docks the markers + whole Group-B stack automatically — touch the per-block offset constants only if a CARD's height changes. Don't flatten the chain back to literals. MOBILE (`@media max-width:767px`) is NOT on the chain — it still hardcodes these px values, so there the old "remeasure the linked set together, per viewport" rule still applies.
- `.about-card--short` has no min-height by design; the hero docks against its natural bottom via hand-tuned `--hero-top` per viewport — if the copy grows, bump `--hero-top` by the height delta; never ship a hardcoded min-height without a retuned `--hero-top`.

- Sheet harmonization (Phase 3) — content COUPLED + CENTRED to the frame, **iPad+desktop only** (`@media (min-width:768px) and (min-height:501px)`, the exact complement of the mobile block, so mobile keeps its full-bleed composition; base rules carry no Phase-3 changes). - Expand-dissolve: the canvas cross-fades filled→line (and back) on expand/collapse. The `expanded` prop → `field.setExpanded(on)` sets `expandTarget`; `expandAmt` eases on a TIME-based smootherstep over `EXPAND_DUR` (800ms) — NOT a per-frame exponential (that front-loaded the change and "popped" the fills back in on collapse). Two bakes — `settled` (filled) + `settledLines` (`--surface-bg` ground + terra stroke, both read at mount into `lineGround`/`lineStroke`) — blitted with the line on top at `globalAlpha = expandAmt` (shared `linePath`, so the wireframe holds while only fills fade). While expanded (`expandTarget===1`) `pointer()` early-returns so the field is a quiet backdrop (hover lamps / ripples / void-rupture suppressed); `setExpanded(true)` calls `resetFocus()`. Full rationale: `ANOMALIES.md` → "Expand-dissolve".
- `.startooth-canvas-root` must stay `position: fixed` — `absolute` causes the canvas to scroll away when the landing expands, leaving an empty background behind the lower Group B cards.
- The CSS `transition` for the build-gate fade must live only in `html.fonts-ready .landing` (the base rule, 0,0,2,1 specificity) — never add a `transition` override to the `:not(.landing--built)` guard rule, or the before-change state will have `transition: none` and the reveal will snap instead of fade.
- The first-paint background is TWO-LAYER and BREAKPOINT-SPLIT: `html:has(.landing), html:has(.landing) body { background-color: #000 }` in `landing.css` (pre-canvas-mount) + `background: #000` on `.startooth-canvas-root` in `startooth-canvas.css` (pre-first-draw) — removing either flashes. On DESKTOP (`@media (min-width:768px) and (min-height:501px)`, framed-sheet complement) a body/`:has()` override swaps first paint to `var(--workbench-bg)` (grey desk) so the sheet arrives on the desk with no black flash, then inks to dark grey (`--grey-80`; see the staged-sheet-intro line); MOBILE keeps the black `#000` base + built-swap (byte-identical). Don't remove the desktop override or unify the two — the black first-paint is mobile-only now.
- Staged sheet intro (DESKTOP-ONLY, gated to the framed-sheet complement `@media (min-width:768px) and (min-height:501px)`; "Staged intro" block in `startooth-canvas.css` + a JS mount delay in `StartoothCanvas.tsx`): the sheet PLACES itself, holds grey, inks at build-START, frame last. Sequence: bare grey desk → the card places (both `.startooth-canvas-root` + `.startooth-sheet-matte` run `@keyframes sheet-place` opacity 0→1, `scale` 0.94→1, `rotate` -2deg→0 on `animation … var(--dur-place) … 0.2s both`; individual `scale`/`rotate` props NOT `transform` so they COMPOSE with the centring `translateX`; 0.2s delay + `both` = desk shows alone first; rounded `border-radius:12px`, grey hairline, matte `border-color:transparent`) → holds grey a beat (the engine's own `BUILD_HOLD` draws its dark ground (`--grey-80`, not the light desk grey), so grey can't come from the engine — `StartoothCanvas` DELAYS `field.mount()` by `GREY_HOLD_MS` on a fresh DESKTOP load, gated `!skipBuild` + `matchMedia('(min-width:768px) and (min-height:501px)')`; mobile/skip keep the original idle-slot mount) → build CUED: `onBuildStart` → `.landing--building` inks grey→dark-grey + straightens corners `12px`→`0` on root + matte (REPLACES the old `.landing--built` trigger). The ink is the CANVAS element (`.startooth-canvas-root canvas`) easing `opacity` 0→1 over the grey root (held `0` through the grey hold) — NOT a `background-color` swap: the canvas paints its dark ground (`--grey-80`, `this.ground`) opaquely on top, so a bg transition is invisible and the cut reads abrupt; the old `@keyframes sheet-ink` was REMOVED. Reduced motion shows the canvas at `opacity:1` (no fade) → objects + caption reveal (landing.css) → frame LAST (matte `border-color` transparent→`var(--surface-bg)` at `.landing--frame`, root grey hairline `box-shadow` retires then, never a double-frame). Keyed off landing stage classes via `html:has()` (matte + root are SIBLINGS of `.landing`). - Full-bleed mobile-Safari edges need TWO things, not just one: (a) `viewport.viewportFit = 'cover'` (root `app/layout.tsx`, GLOBAL — landing has no own layout) so the layout viewport spans the screen; AND (b) a page-background swap, because iOS paints the status-bar/toolbar backdrop with the PAGE bg (not the fixed canvas it can't render there) — so `html:has(.landing--built)` + its `body` swap from the first-paint `#000` to `var(--workbench-bg)` (the `/all` desk colour, matching the framed-sheet matte margins — retargeted from `--surface-bg` when the canvas became the framed sheet) at build-complete, blending the bands into the settled field. The swap is gated on `.landing--built` (NOT first paint) because the canvas is `ssr:false`; recolouring before it mounts would flash. Don't move the swap to first paint, and don't expect cover alone to fix the bars. Full rationale + the CaptionTag note: `ANOMALIES.md` → "Full-bleed canvas needs viewport-fit: cover".
- `html.fonts-ready .landing { pointer-events: none }` in `landing.css` overrides the globals.css auto rule so the canvas receives events in empty areas; the opt-in is scoped to the interactive ELEMENTS `.hero-card`, `.landing-nav-row`, `.caption-tag` — NOT their sections. `.landing__section--hero`/`--projects` boxes run far taller than their content (hero measured 132→878px), so opting the SECTIONS in made empty area swallow clicks meant for the docked tabs + cards below; don't revert the opt-in to the section selectors. `.caption-tag` must be in the list because it's a DOM descendant of `.landing` and inherits pointer-events through the tree despite being `position: fixed`.
- The `landing--built` class has three triggers: `onBuildComplete` from the canvas engine (also sets the module-level `builtThisLoad`), a `useLayoutEffect` that reads it at mount to skip the build, and a `FAILSAFE_MS` (`page.tsx`, ~10s) `setTimeout` failsafe — removing the failsafe strands users whose canvas fails to initialize at a black screen. `FAILSAFE_MS` must clear the desktop grey-hold (`GREY_HOLD_MS`) + build-complete, so the two track together (both were trimmed faster in step). Separate from `onBuildComplete`, `onBuildStart` (sibling callback) fires the instant the grey-held mount is cued → `handleBuildStart` sets `buildStarted` → `.landing--building` (inks the sheet + straightens corners at build START).
- Build-skip memory is a MODULE-LEVEL `builtThisLoad`, never `sessionStorage` — it must reset on hard refresh (build replays) yet persist across client-side nav (return from /selected stays settled). sessionStorage survives the refresh and would wrongly skip every reload.
- Hero headline cycling (`HERO_HEADLINES`, `headlineIdx`) uses `localStorage` — the INVERSE persistence need of `builtThisLoad` above (must survive a hard refresh, unlike the build-skip flag) — and swaps the index in a `useLayoutEffect` off a `useState(0)` default (not a lazy initializer), so SSR/first client render always agree on headline A before the pre-paint swap; a lazy initializer reading localStorage would hydration-mismatch.
- `StartoothField` keeps `prefersReduced` (matchMedia, permanent) separate from `reduced` (per-build collapse = `prefersReduced || skipBuild`); `rebuildFrom(x,y)` (the re-grow seam, no UI yet — for the deferred void rupture) resets `reduced` to `prefersReduced` alone and drops the `HOLD`, so a rebuild always animates (even on a skip-loaded page) instead of running instantly. Don't re-merge the two flags.
- `stepFocus()` hover-release reworks the handoff's jerky release three ways (all deliberate — don't restore the handoff): fade from captured `fadeFrom` not `1 - t` (else snaps to full on sub-0.66s hovers); keep the released key lit via `fadeUnit` (`drawInteractive` reads `focusUnit ?? fadeUnit`) so the lamp dims instead of snapping dark; and `LINGER` + `smoother(t)` so the fade lingers then eases out with no jerk.
- Staged entrance is gated stages, not a block fade: (1) build fires at `prog > 0.76` (on-screen fills done ~0.79; off-screen tilt corners finish last; `bakeSettled` is prog-independent) so the card reveals as the LAST visible corner paints — they overlap, no pop; don't push back toward 0.85+; (2) the card runs `hero-card-settle` over `--dur-place` on `.landing--built:not(.landing--skip):not(.landing--slide-in)` — **rise-then-sit** with a small, deliberately-shallow overshoot above rest (an authored, documented deviation from "no overshoot") and shadow flat→`--shadow-raised`, opacity NOT in the keyframe (the block reveal owns it at `--dur-fast`, so opacity is fast while the motion is slow); (3) about-cards are `opacity:0` via `.landing--built:not(.landing--staged):not(.landing--expanded) .about-card` so they don't bleed through the fading hero; (4) `staged` (separate from `built`, fires `STAGE_SETTLE_MS` — ≈ `--dur-place` — later via the `page.tsx` setTimeout) adds `.landing--staged`, releasing the nav row's `works-tuck-out` (paused until then) and the about-cards; (5) the caption is its own stage — `captionIn` flips `REVEAL_BEAT_MS` after `staged` (a second setTimeout) and gates the caption's `visible` (`!expanded && captionIn`), so it slides up a beat after the tabs, not with them; (6) `frameIn`/`.landing--frame` is the LAST beat (desktop) — a `frameIn` state flips a further `REVEAL_BEAT_MS` after `captionIn` (a third setTimeout; caption + frame share the ONE uniform beat) and reveals the brown sheet frame after the caption (see the staged-sheet-intro line for the intro visuals). Skip path sets `frameIn` synchronously with built/staged/captionIn in the `useLayoutEffect`. Each entrance is `animation-play-state: paused` until its trigger or it plays behind the build curtain. Retune `--dur-place`, the `staged` delay, the `captionIn` delay, and the `frameIn` delay together.
- `.landing--skip` (added with `.landing--built` on session-return) sets `transition: none` on the landing — the skip path flips `built` before first paint, so a reveal transition would arm from a never-painted from-state and PIN at `currentTime:0`, stranding returning visitors at `opacity:0`. Never drop `.landing--skip` or give the skip path a reveal transition.
- Changing the landing's icon set: run `npm run icons`; if it fails with a `pyexpat` import error (Homebrew python3.14 breakage), fall back to `/usr/bin/python3 scripts/icon_subset.py` — the pre-push hook checks the subset and will block a stale one.
- Idle breathing (spec in `DESIGN.md` → "Idle breathing"): after `IDLE_BREATH` (~9s) untouched, random key-tops swell gently via the `topScatter` path (low `mag`, `swell` envelope), driven by the self-scheduling `idleTick`/`scheduleIdle` timer (NOT a continuous rAF — it kicks the loop only per breath). `lastActivity` (pointer + build-complete) resets the idle clock; skipped under reduced motion; `idleTimer` cleared in `destroy()`.
- Void rupture (hidden 9-click easter egg; spec in `DESIGN.md`, constraints in `ANOMALIES.md` → "Void rupture"): a void = star or diamond (`!isKey(u)`); the `rupturing` branch leads `loop()` and `advancePalette()` runs before `rebuildFrom` (build/bake read `COL`); charge is timestamp-derived via `chargeLive()` (never per-frame); `chargeVoid` is a live `Unit` pointer cleared on every lattice rebuild (`start()` + `resize()`); the break is a precomputed pulled-plug flicker, not a dissolve; `onBuildComplete` must stay idempotent (it re-fires on every regrow). Don't reorder the loop branch or the palette/rebuild pair.
- `build()`'s lattice tile range is origin-relative and asymmetric (reaches each canvas edge independently, e.g. `iLo = floor((0-cx)/(PERX*ss))-2` … `iHi = ceil((W-cx)/(PERX*ss))+2`) — never simplify it to a symmetric ±count, which only reaches both edges from a centre origin and leaves corner-origin rebuilds (void ruptures, via `rebuildFrom`) with a far-side black wedge. Same iteration count, so no perf reason to revert; the bug stays invisible on first load (always centre-origin).
- Rupture haptic (Android; iOS no-op) is `flickerSchedule` mapped segment-for-segment to `navigator.vibrate` — keep it derived from the schedule (the on→off→on alternation IS the `[buzz, pause, …]` contract) so buzz can't drift from the visual; the per-charge wind-up scales pulse length (no amplitude in the API); both guarded by `!prefersReduced && navigator.vibrate`. See `ANOMALIES.md` → "Void rupture".
- On resize `lockedKeys` is cleared (same as `chargeVoid`) — both hold live `Unit` pointers from the old lattice; iOS address-bar toggle triggers this path.
- Touch presses on keys activate the lamp effect via `pressFocus` on the `pointerType === 'touch'` branch in `pointer()` — do not remove or unify it; it's the touch substitute for hover.
- On slow-connection mobile, the failsafe may reveal the landing before the canvas build completes; the build then animates behind the settled content — expected degradation, not a bug; don't lower the failsafe timer.
- `.contact-card__form-reveal` carries both `aria-hidden` and `inert` when closed — `aria-hidden` alone doesn't block keyboard focus; don't remove `inert`.
