## Startooth engine vocabulary

**Voids** — `star` (8-sided) and `diamond` (4-sided, below the star). `isVoid(u)` checks both.
**Keys** — `shortkey` (upper, between two stars; FACES_SHORTKEY + TOP_SHORTKEY) and `tallkey` (lower, between star and diamond; FACES_TALLKEY + TOP_TALLKEY). `isKey(u)` checks both. Key parts: **faces** (left+right panels) and **top** (protruding cap, distinctly coloured).
**Static data** — all geometry, palette, and timing live in `startooth-constants.ts`. `StartoothField` holds them as `private readonly` aliases — method bodies are unchanged. Editing a constant: change it in `startooth-constants.ts` only.

## Don't-touch digest

- Two motion idioms must never mix: Group A (about-short, about-long) hero-tucks via top/scale transitions; Group B (spectrum, about-practice, contact) settles from above via opacity + translateY — and about-practice is Group B, not an extension of about-long, even though the copy continues.
- `--stack-stagger-start` is tuned against Group A's tuck-out duration; lowering it makes the two groups visually compete.
- Group B cascade delays are sequential (spectrum +0s, about-practice +0.02s, contact +0.04s) — any new Group B card continues the cascade, never restarts at 0.
- Every Group B card in `.landing--default` must hold all four properties: top at its final expanded position, opacity 0, pointer-events none, and the `translateY(calc(-1 * var(--stack-settle)))` offset — dropping any one breaks the settle-from-above physics.
- Never re-anchor spectrum or contact's collapsed state to `--hero-top` or reintroduce `scale()` on them — that recreates the ghost-cards-descending bug.
- Group B cards rest at 0deg; reintroducing random rotation requires BOTH restoring `@property <angle>` typing AND batching the reroll with `setExpanded` in the same React commit.
- Spectrum's fixed `rotate(-1deg)` is an authored artistic tilt preserved at all breakpoints — it is not part of the removed random-rotation system; don't strip it when enforcing the 0deg rule.
- On expand, each Group B card's top, opacity, and transform transition together with the same duration and delay on desktop — never stagger the three within one card. (Documented mobile deviation: spectrum's transform rides `--dur-settle` +0.12s — shipped and authored; don't "fix" it or copy it to other cards.)
- The form-open height bump (`:has(.contact-card--form-open)`, +600px) must be remeasured at all breakpoints whenever `--contact-top`, the contact card's collapsed height, or the form's reveal target changes.
- `--expanded-h`, `--spectrum-top`, `--practice-top`, and `--contact-top` are a linked set anchored to measured card heights — remeasure dependent tops together, per viewport, never one in isolation.
- `.about-card--short` has no min-height by design; the hero docks against its natural bottom via hand-tuned `--hero-top` per viewport — if the copy grows, bump `--hero-top` by the height delta; never ship a hardcoded min-height without a retuned `--hero-top`.
- `--long-top` and `--projects-top` move with `--hero-top` by the same delta per viewport, and the hero-overlap split (33px desktop vs 9px mobile) is intentional — do not unify the two figures.
- about-long's horizontal padding is `--space-24` (not the standard 32) so the longest discipline row + chip fits on one line — widen only behind a viewport guard.
- about-long's copy is centered by design intent; right-edge/tabular chip alignment was tried and rejected ("paying too much attention to the numbers") — do not re-propose it.
- The `whiteSpace: nowrap` span around "growth experiments" in app/page.tsx row 3 is load-bearing for the wrap shape — keep it when editing copy.
- Mobile about-short intentionally drops the bottom divider and bottom padding (above-hero space is tight) — don't restore without re-measuring and bumping `--hero-top`.
- The landing scrollbar is hidden (`html:has(.landing)`) because the expand-on-click affordance carries the "more content" signal — reconsider only if that affordance changes.
- Desktop collapsed about-long is hand-tuned to stay fully behind the hero (`top: calc(var(--hero-top) - 20px)`, scale 0.82) while mobile keeps `top: var(--hero-top)` at scale 0.9 — re-measure both offsets if the card's copy grows; don't unify the two recipes.
- `.startooth-canvas-root` must stay `position: fixed` — `absolute` causes the canvas to scroll away when the landing expands, leaving an empty background behind the lower Group B cards.
- The CSS `transition` for the build-gate fade must live only in `html.fonts-ready .landing` (the base rule, 0,0,2,1 specificity) — never add a `transition` override to the `:not(.landing--built)` guard rule, or the before-change state will have `transition: none` and the reveal will snap instead of fade.
- The black first-paint background requires two layers: `html:has(.landing) body { background-color: #000 }` in `landing.css` (pre-canvas-mount) and `background: #000` on `.startooth-canvas-root` in `startooth-canvas.css` (pre-first-draw) — removing either creates a visible flash.
- `viewport.viewportFit = 'cover'` (root `app/layout.tsx`, GLOBAL) is what lets the fixed canvas reach the true mobile-Safari edges; without it the `#000` body shows as black bars in the status-bar / home-indicator safe areas. Global because the landing has no own layout (client page under root). Trade-off + why the bottom `CaptionTag` is intentionally left unguarded: see `ANOMALIES.md` → "Full-bleed canvas needs viewport-fit: cover".
- `html.fonts-ready .landing { pointer-events: none }` in `landing.css` overrides the globals.css auto rule so the canvas receives events in empty areas; `.landing__section--hero`, `.landing__section--projects`, and `.caption-tag` explicitly opt back in — `.caption-tag` must be in this list because it is a DOM descendant of `.landing` and inherits pointer-events through the tree despite being `position: fixed`.
- The `landing--built` class has three triggers: `onBuildComplete` from the canvas engine (also sets the module-level `builtThisLoad`), a `useLayoutEffect` that reads it at mount to skip the build, and an 8s `setTimeout` failsafe in `page.tsx` — removing the failsafe strands users whose canvas fails to initialize at a black screen.
- Build-skip memory is a MODULE-LEVEL `builtThisLoad`, never `sessionStorage` — it must reset on hard refresh (build replays) yet persist across client-side nav (return from /selected stays settled). sessionStorage survives the refresh and would wrongly skip every reload.
- `StartoothField` keeps `prefersReduced` (matchMedia, permanent) separate from `reduced` (per-build collapse = `prefersReduced || skipBuild`); `rebuildFrom(x,y)` (the re-grow seam, no UI yet — for the deferred void rupture) resets `reduced` to `prefersReduced` alone and drops the `HOLD`, so a rebuild always animates (even on a skip-loaded page) instead of running instantly. Don't re-merge the two flags.
- `stepFocus()` hover-release reworks the handoff's jerky release three ways (all deliberate — don't restore the handoff): fade from captured `fadeFrom` not `1 - t` (else snaps to full on sub-0.66s hovers); keep the released key lit via `fadeUnit` (`drawInteractive` reads `focusUnit ?? fadeUnit`) so the lamp dims instead of snapping dark; and `LINGER` + `smoother(t)` so the fade lingers then eases out with no jerk.
- Staged entrance is gated stages, not a block fade: (1) build fires at `prog > 0.76` (on-screen fills done ~0.79; off-screen tilt corners finish last; `bakeSettled` is prog-independent) so the card reveals as the LAST visible corner paints — they overlap, no pop; don't push back toward 0.85+; (2) the card runs `hero-card-settle` over `--dur-place` on `.landing--built:not(.landing--skip):not(.landing--slide-in)` — **rise-then-sit** with a small, deliberately-shallow overshoot above rest (an authored, documented deviation from "no overshoot") and shadow flat→`--shadow-raised`, opacity NOT in the keyframe (the block reveal owns it at `--dur-fast`, so opacity is fast while the motion is slow); (3) about-cards are `opacity:0` via `.landing--built:not(.landing--staged):not(.landing--expanded) .about-card` so they don't bleed through the fading hero; (4) `staged` (separate from `built`, fires ≈1300ms later via the `page.tsx` setTimeout) adds `.landing--staged`, releasing the nav row's `works-tuck-out` (paused until then) and the about-cards; (5) the caption is its own stage — `captionIn` flips ~700ms after `staged` (a second setTimeout) and gates the caption's `visible` (`!expanded && captionIn`), so it slides up a beat after the tabs, not with them. Each entrance is `animation-play-state: paused` until its trigger or it plays behind the build curtain. Retune `--dur-place`, the `staged` delay, and the `captionIn` delay together.
- `.landing--skip` (added with `.landing--built` on session-return) sets `transition: none` on the landing — the skip path flips `built` before first paint, so a reveal transition would arm from a never-painted from-state and PIN at `currentTime:0`, stranding returning visitors at `opacity:0`. Never drop `.landing--skip` or give the skip path a reveal transition.
- Changing the landing's icon set: run `npm run icons`; if it fails with a `pyexpat` import error (Homebrew python3.14 breakage), fall back to `/usr/bin/python3 scripts/icon_subset.py` — the pre-push hook checks the subset and will block a stale one.
- Idle breathing (spec in `DESIGN.md` → "Idle breathing"): after `IDLE_BREATH` (~9s) untouched, random key-tops swell gently via the `topScatter` path (low `mag`, `swell` envelope), driven by the self-scheduling `idleTick`/`scheduleIdle` timer (NOT a continuous rAF — it kicks the loop only per breath). `lastActivity` (pointer + build-complete) resets the idle clock; skipped under reduced motion; `idleTimer` cleared in `destroy()`.
- Void rupture (hidden 9-click easter egg; spec in `DESIGN.md`, constraints in `ANOMALIES.md` → "Void rupture"): a void = star or diamond (`!isKey(u)`); the `rupturing` branch leads `loop()` and `advancePalette()` runs before `rebuildFrom` (build/bake read `COL`); charge is timestamp-derived via `chargeLive()` (never per-frame); `chargeVoid` is a live `Unit` pointer cleared on every lattice rebuild (`start()` + `resize()`); the break is a precomputed pulled-plug flicker, not a dissolve; `onBuildComplete` must stay idempotent (it re-fires on every regrow). Don't reorder the loop branch or the palette/rebuild pair.
- `build()`'s lattice tile range is origin-relative and asymmetric (reaches each canvas edge independently, e.g. `iLo = floor((0-cx)/(PERX*ss))-2` … `iHi = ceil((W-cx)/(PERX*ss))+2`) — never simplify it to a symmetric ±count, which only reaches both edges from a centre origin and leaves corner-origin rebuilds (void ruptures, via `rebuildFrom`) with a far-side black wedge. Same iteration count, so no perf reason to revert; the bug stays invisible on first load (always centre-origin).
- Rupture haptic (Android; iOS no-op) is `flickerSchedule` mapped segment-for-segment to `navigator.vibrate` — keep it derived from the schedule (the on→off→on alternation IS the `[buzz, pause, …]` contract) so buzz can't drift from the visual; the per-charge wind-up scales pulse length (no amplitude in the API); both guarded by `!prefersReduced && navigator.vibrate`. See `ANOMALIES.md` → "Void rupture".
- On resize `lockedKeys` is cleared (same as `chargeVoid`) — both hold live `Unit` pointers from the old lattice; iOS address-bar toggle triggers this path.
- Touch presses on keys activate the lamp effect via `pressFocus` on the `pointerType === 'touch'` branch in `pointer()` — do not remove or unify it; it's the touch substitute for hover.
- On slow-connection mobile, the 8s failsafe may reveal the landing before the canvas build completes; the build then animates behind the settled content — expected degradation, not a bug; don't lower the failsafe timer.
- `.contact-card__form-reveal` carries both `aria-hidden` and `inert` when closed — `aria-hidden` alone doesn't block keyboard focus; don't remove `inert`.
